reconstruction → 3DGS ply.

## Our config (12 GB card, decided from the README VRAM table)

- SKIP their Text2PanoImage stage (~16 GB VRAM > 12): README explicitly
  supports user-supplied panos ("generated by other methods like
  HunyuanWorld") — drop `pano_img.jpg` + `prompt.txt` in the run dir, start
  at step 2. Our panos: out/bedroom_hw1/panorama.png etc.

  Wan2.2-TI2V-5B base + pano LoRA) = ~12 GB VRAM. ~1 h on an A800 ⇒
  overnight-class on the 4080 Laptop. RUNTIME = GUARDED (deadman, caps).
- Recon: **optimization-based** (~10 GB VRAM); PanoLRM (~80 GB) skipped.
